<Section title="Getting the flag" />
<Paragraph>Another hint! the flag must be in the file name, but where? wait a sec, let me do a mini research...</Paragraph>
<Paragraph>Okay so.... from what i found. Analyzing a filename in Linux disk forensics involves mapping the human-readable string to its corresponding filesystem structure (like an inode or directory entry) and analyzing associated metadata</Paragraph>
<Paragraph>Which mean, there should be inode that exists specifically for filename, and yeah, i found it.</Paragraph>
<Codeblock filename="specialExt4Inode.txt">
```
Inode 2: The Root Directory (/) of the filesystem.
Inode 1: List of defective/bad blocks.
Inode 7: The boot loader block tracking.
Inode 8: Journal data tracking.
Inode 11: The traditional first available inode for normal files/directories (often utilized by lost+found).
```
</Codeblock>
<Paragraph>This is a common special inode on Ext4, and fortunately, the given partition is Ext4</Paragraph>
<Codeblock filename="partitionInformation.txt">
