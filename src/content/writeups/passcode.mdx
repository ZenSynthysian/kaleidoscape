<Paragraph>notice welcome is returned before login get called? that behaviour may can lead to stack frame reuse, because the stack frame used by welcome() is later reused by login(), the last few bytes of the name buffer overlap with the memory that will later hold passcode1. By supplying a sufficiently long input, I can control the initial value of passcode1. Since scanf("%d", passcode1) treats that value as a pointer, I can control the address that scanf will write to.</Paragraph>
<br/>
<Paragraph>look at the highlighted code, you can see the name variable is -0x70 from rbp and passcode1 -0x10 from rbp, to calculate how many byte needed to reach passcode 1, you can substract 0x70 with 0x10, so we need 96 byte to reach passcode1 from name variable</Paragraph>

<Paragraph>from the highlighted line, the best candidate for overwrite is `.got.plt`, why?</Paragraph>
<br/>
<Paragraph>`Global Offset Table` or known as `GOT` is a section where the absolute memory addresses of external symbols<i>(like imported functions)</i> are stored.</Paragraph>
<br/>
<Paragraph>here how it works. programmer import function from external source -> the program use .plt, dynamic addressing for the function used, often unwriteable -> .plt has .got where absolute address of imported function stored, program will jump to this address and run the function</Paragraph>
<br/>
<Paragraph>if we somehow can overwrite the `GOT address` inside `.plt` function, we can jump to the address we write, and from the code below, the nearest function will be called is `fflush()`, so our objective now is overwrite the got address of fflush inside the .plt section, usually located in the very first line of .plt</Paragraph>

```python
io = sh.shell("./passcode")
payload = b"a"*96
payload += p32(0x0804c014)
payload2 = b"134517401"
io.sendline(payload)
io.sendline(payload2)
print(io.recvall(3))
```
